# Use Case-05.05: Validation AI Questions

**Use Case ID:** UC-05.05

**Module:** Tính Năng AI (AI Features)

**Actor chính:** Hệ thống (System)

**Actor phụ:** Không có

**Mô tả ngắn:** Validate câu hỏi AI-generated trước khi show cho instructor

**Trigger:** Backend nhận response từ Gemini API (từ UC-05.03 hoặc UC-05.04)

**Pre-condition:**

- Gemini API đã trả về JSON array of questions
- Questions chưa được validate

**Post-condition:**

- Questions passed validation được return
- Questions failed validation bị reject và regenerate
- Success rate được log vào database

**Main Flow:**

1. System nhận JSON response từ Gemini API
2. System parse JSON thành question objects
3. Đối với mỗi question, system validate:

   **Check structure:**

   - Có đúng 1 correct answer
   - Có 3-4 incorrect options (total 4-5 options)
   - Tất cả options non-empty

   **Check clarity:**

   - Question length: 10-500 characters
   - Không có duplicate options

   **Check difficulty alignment:**

   - Verify generated difficulty tag (Easy/Medium/Hard)
   - Count số câu Easy, Medium, Hard
   - So với requested ratio (±5% tolerance)

4. Nếu question PASS tất cả checks:

   - Add vào valid_questions array
   - Đánh dấu validation_status = "passed"

5. Nếu question FAIL bất kỳ check nào:

   - Add vào failed_questions array
   - Log lý do fail (structure/clarity/difficulty)

6. System kiểm tra difficulty alignment:

   - Actual ratio = (Easy_count : Medium_count : Hard_count)
   - Requested ratio (ví dụ: 30:50:20)
   - Tolerance = ±5%
   - Nếu vượt tolerance: mark cần regenerate

7. Nếu có failed_questions:

   - Auto-regenerate failed questions (gọi lại Gemini API)
   - Max 3 retry attempts
   - Mỗi retry chỉ regenerate số câu failed

8. Sau retries:

   - Nếu vẫn có questions fail: accept current valid_questions
   - Log warning vào database

9. System tính success rate:

   - success_rate = valid_count / total_requested × 100%

10. System log vào database table ai_generation_logs:

    - timestamp
    - instructor_id
    - document_ids
    - total_requested
    - total_valid
    - success_rate
    - retry_count

11. System return valid_questions array về controller
12. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Tất cả questions fail validation sau 3 retries**

  - Tại bước 8: valid_questions array empty
  - System return error: "Không thể generate câu hỏi hợp lệ. Vui lòng thử lại."
  - Log error vào database
  - Kết thúc Use Case

- **AF2: Difficulty ratio vượt quá tolerance**

  - Tại bước 6: Actual ratio khác requested >5%
  - Ví dụ: request 30:50:20 nhưng nhận được 40:40:20
  - System regenerate một số câu để cân bằng ratio
  - Max 2 attempts để balance
  - Nếu vẫn không balance: accept và log warning

- **AF3: Duplicate questions phát hiện**

  - Tại bước 3: Question text giống hệt question khác trong batch
  - Mark as failed với lý do "duplicate"
  - Regenerate question đó

- **AF4: Invalid JSON response từ Gemini**
  - Tại bước 2: JSON parse error
  - System retry entire request (max 3 times)
  - Nếu vẫn fail: return error "Invalid AI response"
  - Kết thúc Use Case
