# Use Case-04.08: Tạo Assignment

**Use Case ID:** UC-04.08  
**Module:** Hệ Thống Quiz & Assignment  
**Actor chính:** Instructor  
**Actor phụ:** Không có

**Mô tả ngắn:** Instructor tạo bài tập yêu cầu nộp file hoặc văn bản

**Trigger:** Instructor nhấn "Create Assignment"

**Pre-condition:**

- Instructor đã đăng nhập
- Instructor sở hữu khóa học

**Post-condition:**

- Assignment mới được tạo
- Học viên có thể bắt đầu nộp bài

**Main Flow:**

1. Instructor nhấn "Add Assignment" trong course builder
2. Hệ thống hiển thị form nhập liệu
3. Instructor điền thông tin:
   - Tiêu đề (tối đa 200 ký tự)
   - Hướng dẫn (rich text)
   - Hạn nộp (ngày và giờ)
   - Tổng điểm
   - Loại nộp: File Upload, Text Entry, hoặc Code Submission
   - Nếu File Upload: loại file (PDF/DOCX/ZIP), dung lượng tối đa (5-50MB)
   - Nếu Code: ngôn ngữ (Python/Java/C++/JavaScript)
   - Cho phép nộp muộn (Có/Không)
   - Nếu Có: Mức phạt nộp muộn (% mỗi ngày)
   - Phương thức chấm: Thủ công, Hỗ trợ AI, hoặc Hỗn hợp
4. Hệ thống xác thực:
   - Tiêu đề không trống
   - Hạn nộp sau thời gian hiện tại
   - Điểm số > 0
   - Dung lượng file 5-50MB
5. Hệ thống tạo bản ghi assignment
6. Hệ thống thông báo: "Bài tập đã được tạo"
7. Hệ thống gửi notification đến học viên đã enroll
8. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Hạn nộp trong quá khứ**

  - Tại bước 4: Hạn nộp < thời gian hiện tại
  - Hệ thống hiển thị lỗi: "Hạn nộp phải là thời gian tương lai"
  - Quay lại bước 3

- **AF2: Mức phạt nộp muộn > 100%**
  - Tại bước 4: Mức phạt > 100%
  - Hệ thống cảnh báo: "Mức phạt trên 100% sẽ khiến điểm âm"
  - Cho phép tiếp tục nhưng hiển thị cảnh báo
