# Use Case-04.09: Nộp Assignment

**Use Case ID:** UC-04.09  
**Module:** Hệ Thống Quiz & Assignment  
**Actor chính:** Student  
**Actor phụ:** Không có

**Mô tả ngắn:** Student nộp bài làm cho assignment

**Trigger:** Student nhấn "Submit Assignment"

**Pre-condition:**

- Student đã enroll khóa học
- Assignment tồn tại và đang mở

**Post-condition:**

- Submission được lưu
- Instructor nhận thông báo

**Main Flow:**

1. Student điều hướng đến trang assignment
2. Hệ thống hiển thị chi tiết assignment và form nộp
3. Student thực hiện nộp theo loại:
   - **File Upload:** Upload file (drag-drop hoặc chọn từ máy)
   - **Text Entry:** Nhập nội dung vào rich text editor
   - **Code:** Dán code vào code editor (có syntax highlighting)
4. Hệ thống xác thực:
   - Định dạng file (nếu File Upload)
   - Dung lượng file (≤ max size)
   - Nếu Code: Kiểm tra syntax sơ bộ
5. Hệ thống hiển thị tiến trình upload (nếu là file)
6. Hệ thống upload file lên S3 (nếu File Upload)
7. Hệ thống tạo submission record:
   - student_id
   - assignment_id
   - submission content (URL hoặc text)
   - submitted_at (timestamp)
   - is_late (nếu submitted_at > due_date)
8. Hệ thống gửi email xác nhận cho Student
9. Hệ thống gửi notification cho Instructor
10. Hệ thống hiển thị: "Nộp bài thành công! Đã nộp lúc [timestamp]"
11. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Định dạng file không hợp lệ**

  - Tại bước 4: File không nằm trong danh sách cho phép
  - Hệ thống hiển thị lỗi: "Chỉ chấp nhận [danh sách định dạng]"
  - Quay lại bước 3

- **AF2: File quá lớn**

  - Tại bước 4: Dung lượng > max size
  - Hệ thống hiển thị lỗi: "Dung lượng vượt quá [max size]MB"
  - Quay lại bước 3

- **AF3: Lỗi syntax code**

  - Tại bước 4: Code có lỗi syntax
  - Hệ thống cảnh báo: "Code có lỗi syntax, vẫn muốn nộp?"
  - Nếu Đồng ý → Tiếp tục
  - Nếu Không → Quay lại bước 3

- **AF4: Nộp muộn**

  - Tại bước 7: submitted_at > due_date
  - Hệ thống đánh dấu is_late = true
  - Áp dụng mức phạt (khi chấm điểm)
  - Hiển thị: "Nộp muộn! Mức phạt [X]% mỗi ngày"

- **AF5: Nộp lại trước hạn**

  - Tại bước 7: Student đã có submission trước đó
  - Thời gian hiện tại < due_date
  - Hệ thống xác nhận: "Thay thế bài nộp cũ?"
  - Nếu Đồng ý → Ghi đè
  - Nếu Không → Hủy

- **AF6: Không cho phép nộp muộn**
  - Tại bước 4: submitted_at > due_date VÀ allow_late = false
  - Hệ thống lỗi: "Đã quá hạn nộp. Không tiếp nhận bài mới"
  - Kết thúc Use Case
