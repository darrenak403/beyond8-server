# Use Case-04.06: Chấm Điểm Tự Động Câu Trắc Nghiệm

**Use Case ID:** UC-04.06  
**Module:** Hệ Thống Quiz & Assignment  
**Actor chính:** System  
**Actor phụ:** Không có

**Mô tả ngắn:** Hệ thống tự động tính điểm cho câu hỏi trắc nghiệm

**Trigger:** Student nộp quiz (UC-04.05 bước 11)

**Pre-condition:**

- Student đã nộp quiz
- Quiz có câu hỏi trắc nghiệm

**Post-condition:**

- Điểm số được tính và lưu
- Trạng thái Pass/Fail được xác định

**Main Flow:**

1. Hệ thống nhận quiz submission từ UC-04.05
2. Hệ thống duyệt qua tất cả câu hỏi trắc nghiệm
3. Với mỗi câu hỏi:
   - So sánh đáp án của student với đáp án đúng
   - Nếu đúng: điểm câu = điểm tối đa của câu
   - Nếu sai: điểm câu = 0
4. Hệ thống tính tổng điểm:
   - total_score = SUM(điểm các câu)
5. Hệ thống tính phần trăm:
   - percentage = (total_score / max_possible_score) × 100
6. Hệ thống xác định Pass/Fail:
   - Nếu percentage ≥ passing_percentage → Pass
   - Nếu percentage < passing_percentage → Fail
7. Hệ thống lưu vào quiz_attempt:
   - score
   - percentage
   - status (Pass/Fail)
   - attempt_number
   - submitted_at (timestamp)
8. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Quiz chỉ có câu tự luận**

  - Tại bước 2: Không có câu trắc nghiệm
  - Hệ thống đánh dấu status = "Pending Grading"
  - Lưu attempt với score = NULL
  - Kết thúc Use Case

- **AF2: Quiz có cả trắc nghiệm và tự luận**
  - Tại bước 4: Chỉ tính điểm phần trắc nghiệm
  - Lưu partial_score (điểm tạm)
  - Đánh dấu status = "Partially Graded"
  - Chờ instructor chấm phần tự luận (UC-04.11)
