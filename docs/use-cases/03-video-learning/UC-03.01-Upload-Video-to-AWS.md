# Use Case-03.01: Upload Video to AWS

**Use Case ID:** UC-03.01

**Module:** Học tập qua Video (Video Learning)

**Actor chính:** Giảng viên (Instructor)

**Actor phụ:** AWS S3, CloudFront, FFmpeg/MediaConvert

**Mô tả ngắn:** Giảng viên upload video trực tiếp lên AWS S3 để tạo bài học video

**Trigger:** Instructor nhấn "Upload Video" khi tạo hoặc chỉnh sửa lesson

**Pre-condition:**

- Instructor đã đăng nhập và có quyền tạo/sửa lesson
- Lesson đã được tạo hoặc đang trong quá trình chỉnh sửa
- AWS S3 bucket và CloudFront distribution đã được cấu hình

**Post-condition:**

- Video được lưu trữ trên AWS S3
- Metadata video (duration, resolution, file size) được lưu vào database
- CloudFront URL được sinh tự động
- Lesson status chuyển sang "Processing" cho đến khi video transcoded

**Main Flow:**

1. Instructor click button "Upload Video" trong form tạo/sửa lesson
2. Hệ thống mở upload interface với drag-and-drop hoặc file picker
3. Instructor kéo-thả file video hoặc chọn từ máy tính
4. Hệ thống validate file:
   - Định dạng: MP4, MOV, AVI (bắt buộc, theo BR-06)
   - Kích thước: ≤ 3GB (theo BR-06)
   - Duration: ≤ 30 phút (theo BR-06)
   - Video codec: H.264
   - Audio codec: AAC
5. Hệ thống hiển thị progress bar (0-100%)
6. Hệ thống tạo presigned URL từ AWS S3
7. File được upload lên AWS S3 bucket với:
   - Multipart upload cho file lớn
   - Pause/resume support
8. Sau upload thành công, hệ thống trigger AWS MediaConvert hoặc FFmpeg:
   - Extract metadata: duration, resolution, width, height, file_size
   - Tạo thumbnail từ frame thứ 5 giây hoặc giữa video
   - Tạo HLS/DASH adaptive bitrate streams
9. Hệ thống lưu vào database:
   - video_url (CloudFront URL)
   - duration (giây)
   - file_size (bytes)
   - resolution (1280x720, 1920x1080, etc.)
   - thumbnail_url (CloudFront URL)
   - transcoding_status = "in_progress"
10. Hệ thống hiển thị success notification: "Video uploaded successfully. Processing..."
11. Lesson status = "Processing"
12. Kết thúc Use Case

**Alternate Flow:**

- **AF1: File không đúng định dạng**

  1. Hệ thống validate và từ chối file
  2. Hiển thị error: "Định dạng không hỗ trợ. Vui lòng upload file MP4, WebM hoặc MOV"
  3. Kết thúc

- **AF2: File vượt quá kích thước tối đa**

  1. Hệ thống validate và từ chối file
  2. Hiển thị error: "File quá lớn. Kích thước tối đa 2GB"
  3. Kết thúc

- **AF3: Mất kết nối mạng trong quá trình upload**

  1. Upload bị tạm dừng
  2. Hệ thống hiển thị button "Resume Upload"
  3. Instructor click "Resume Upload"
  4. Tiếp tục upload từ vị trí dừng lại (multipart upload)
  5. Quay lại main flow bước 6

- **AF4: Hủy upload**
  1. Instructor click button "Cancel"
  2. Hệ thống hủy upload và xóa file từ S3
  3. Kết thúc

**Exception Handling:**

- **EH1: AWS API lỗi**

  - Hiển thị error: "Upload failed. Please try again later."
  - Auto-retry tối đa 3 lần
  - Nếu vẫn fail, cho phép instructor retry thủ công

- **EH2: FFmpeg transcoding lỗi**

  - Đánh dấu lesson status = "Transcoding Failed"
  - Gửi notification tới instructor
  - Cho phép instructor re-upload video

- **EH3: Video vượt quá duration limit**
  - Hiển thị error: "Video quá dài. Thời lượng tối đa 30 phút (theo BR-06)"
  - Cho phép instructor re-upload video ngắn hơn

**Business Rules:**

- BR-06: Video limits (max 3GB, max 30 phút duration, MP4/MOV/AVI)
- BR-04: Truy cập & Enrollment (video chỉ xem bởi enrolled students hoặc preview videos)
- Upload phải qua HTTPS secured connection
- Presigned URL hết hạn sau 1 giờ

**Non-Functional Requirements:**

- Upload speed: hỗ trợ download speed của user (adaptive)
- Progress bar update mỗi 5%
- Multipart chunk size: 5MB per chunk
- Timeout cho transcoding: 30 phút

**Postcondition Details:**

- Lesson có thể được lưu trong trạng thái Draft với video đang processing
- Video có thể được preview khi transcoding hoàn thành
- Instructor được notify qua email khi transcoding done
- Video chỉ có thể được xem bởi students đã enrolled (BR-04) hoặc preview videos được đánh dấu
