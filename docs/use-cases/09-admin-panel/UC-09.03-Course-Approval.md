# Use Case-09.03: Course Approval Workflow

**Use Case ID:** UC-09.03  
**Module:** Admin Panel  
**Actor chính:** Admin  
**Actor phụ:** Instructor (nhận notification)

**Mô tả ngắn:** Admin duyệt hoặc từ chối courses do instructor submit

**Trigger:** Admin nhấn "Course Approvals" hoặc quick action từ dashboard

**Pre-condition:**

- Admin đã đăng nhập
- Có khóa học với status = "pending_approval"

**Post-condition:**

- Course được approve (published) hoặc reject
- Instructor nhận notification

**Main Flow:**

1. Admin nhấn "Course Approvals"
2. Hệ thống hiển thị tab "Pending Approval"
3. Hệ thống hiển thị danh sách courses chờ duyệt:
   - Columns: Tiêu đề, Instructor, Danh mục, Ngày gửi, Actions
   - Sắp xếp: Ngày gửi (cũ nhất trước)
4. Admin click vào course để review
5. Hệ thống hiển thị course review interface:
   - Thông tin course: Title, Description, Category, Price
   - Preview sections và lessons
   - Preview videos
   - Preview quizzes/assignments
   - Course curriculum outline
6. Hệ thống hiển thị action buttons:
   - "Approve" (xanh)
   - "Reject" (đỏ)
7. Admin review nội dung khóa học
8. Admin quyết định approve hoặc reject
9. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Admin approve course**

  - Tại bước 8: Admin nhấn "Approve"
  - Hệ thống confirm: "Duyệt và xuất bản khóa học này?"
  - Admin confirm
  - Hệ thống:
    - Cập nhật course.status = "approved"
    - Cập nhật course.approved_at = now
    - Cập nhật course.approved_by = admin_id
    - Publish course (visible to public)
    - Tạo approval log
  - Gửi email cho Instructor: "Khóa học đã được duyệt"
  - Gửi in-app notification cho Instructor
  - Hiển thị: "Khóa học đã được duyệt và xuất bản"
  - Refresh danh sách

- **AF2: Admin reject course**

  - Tại bước 8: Admin nhấn "Reject"
  - Hệ thống hiển thị form:
    - Dropdown "Lý do từ chối":
      - "Vấn đề về chất lượng nội dung"
      - "Vi phạm hướng dẫn"
      - "Thiếu thông tin"
      - "Khác"
    - Nếu chọn "Khác": Text area (bắt buộc, tối đa 500 ký tự)
    - Button "Confirm Rejection"
  - Admin chọn lý do và nhập chi tiết (nếu cần)
  - Admin nhấn "Confirm Rejection"
  - Hệ thống:
    - Cập nhật course.status = "rejected"
    - Lưu rejection_reason, rejected_at, rejected_by
    - Tạo rejection log
  - Gửi email cho Instructor với lý do từ chối
  - Gửi in-app notification
  - Hiển thị: "Khóa học đã bị từ chối"
  - Refresh danh sách

- **AF3: Bulk approval**

  - Tại bước 3: Admin chọn nhiều courses (checkboxes)
  - Admin nhấn "Bulk Approve"
  - Hệ thống confirm: "Duyệt X khóa học đã chọn?"
  - Admin confirm
  - Hệ thống approve tất cả courses đã chọn
  - Gửi notification cho tất cả instructors
  - Hiển thị: "X khóa học đã được duyệt"

- **AF4: Bulk rejection**

  - Tại bước 3: Admin chọn nhiều courses
  - Admin nhấn "Bulk Reject"
  - Hệ thống hiển thị form lý do (chung cho tất cả)
  - Admin nhập lý do và confirm
  - Hệ thống reject tất cả courses với cùng lý do
  - Gửi notification cho tất cả instructors

- **AF5: Xem approval history**
  - Tại bước 5: Admin nhấn "View History"
  - Hệ thống hiển thị approval/rejection history:
    - List tất cả approvals/rejections trước đó
    - Columns: Date, Admin, Action, Reason (if rejected)
  - Admin có thể review lịch sử quyết định
