# Use Case-07.06: Admin Revenue Tracking

**Use Case ID:** UC-07.06

**Module:** Payment & Enrollment

**Actor chính:** Admin

**Actor phụ:** Không có

**Mô tả ngắn:** Admin xem tổng doanh thu của toàn platform

**Trigger:** Admin truy cập Revenue Dashboard trong Admin Panel

**Pre-condition:**

- Admin đã đăng nhập
- Admin có quyền truy cập Admin Panel

**Post-condition:**

- Dashboard doanh thu platform được hiển thị
- Admin có thể xuất toàn bộ dữ liệu

**Main Flow:**

1. Admin navigate đến Admin Panel
2. Admin click "Revenue Tracking" trong menu
3. Hệ thống fetch toàn bộ payment data của platform
4. Hệ thống tính toán thống kê nền tảng:
   - **Tổng doanh thu:** Sum(amount) của tất cả successful payments
   - **Tổng số đăng ký có phí:** Count(enrollments where payment_status = "paid")
   - **Tổng số instructors có thu nhập:** Count(distinct instructors with successful payments)
   - **Giá trị giao dịch trung bình:** Tổng doanh thu / Tổng số giao dịch thành công
5. Hệ thống hiển thị Admin Revenue Dashboard:
   - **Thống kê nền tảng:** 4 cards hiển thị metrics ở bước 4
   - **Xu hướng doanh thu:**
     - Line chart với tùy chọn view: Theo ngày / Theo tuần / Theo tháng
     - Mặc định: Theo tháng (12 tháng gần nhất)
   - **Courses hoạt động tốt nhất:**
     - Table: Tên course, Instructor, Doanh thu, Số đăng ký
     - Top 10 courses theo doanh thu
   - **Instructors có thu nhập cao nhất:**
     - Table: Tên instructor, Tổng doanh thu, Tổng số courses
     - Top 10 instructors
   - **Nhật ký giao dịch:**
     - Table: Transaction ID, Student, Course, Instructor, Amount, Date, Status
     - Pagination: 50 records/page
6. Hệ thống hiển thị bộ lọc:
   - Filter theo Instructor (dropdown, searchable)
   - Filter theo Course (dropdown, searchable)
   - Filter theo khoảng thời gian (date range)
   - Filter theo Status (Success/Failed/Pending)
7. Hệ thống hiển thị button "Xuất tất cả dữ liệu" (CSV/Excel)
8. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Admin filter theo instructor**

  - Tại bước 6: Admin chọn instructor từ dropdown
  - Hệ thống reload data chỉ hiển thị giao dịch liên quan instructor đó
  - Cập nhật tất cả metrics và charts theo filter
  - Kết thúc Use Case

- **AF2: Admin filter theo course**

  - Tại bước 6: Admin chọn course từ dropdown
  - Hệ thống reload data chỉ hiển thị giao dịch cho course đó
  - Cập nhật metrics và charts
  - Kết thúc Use Case

- **AF3: Admin filter theo khoảng thời gian**

  - Tại bước 6: Admin chọn date range
  - Hệ thống reload data với filter: paid_at trong khoảng đã chọn
  - Cập nhật toàn bộ dashboard
  - Kết thúc Use Case

- **AF4: Admin xuất tất cả dữ liệu**

  - Tại bước 7: Admin click "Xuất tất cả dữ liệu"
  - Hệ thống hiển thị dialog chọn format: CSV hoặc Excel
  - Admin chọn format
  - Hệ thống generate file với tất cả transactions (có thể lớn, cần processing time)
  - Hiển thị progress indicator
  - Khi hoàn tất: Tải xuống "Platform_Revenue\_[date].csv"
  - Kết thúc Use Case

- **AF5: Admin thay đổi view biểu đồ**
  - Tại bước 5: Admin click toggle "Theo ngày" / "Theo tuần" / "Theo tháng"
  - Hệ thống reload chart với granularity mới
  - Ví dụ "Theo ngày": hiển thị 30 ngày gần nhất
  - Kết thúc Use Case
