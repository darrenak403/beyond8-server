# Use Case-07.03: VNPay Payment Integration

**Use Case ID:** UC-07.03

**Module:** Payment & Enrollment

**Actor chính:** Student

**Actor phụ:** VNPay Gateway, Email Service

**Mô tả ngắn:** Student thanh toán course qua VNPay gateway

**Trigger:** Student click "Enroll Now" trên paid course và chọn thanh toán

**Pre-condition:**

- Student đã đăng nhập
- Course price > 0
- Course status = "Published"
- Student chưa enroll course

**Post-condition:**

- Payment record được tạo
- Nếu thành công: enrollment được tạo, email xác nhận gửi đi
- Nếu thất bại: Student có thể thử lại

**Main Flow:**

1. Student truy cập trang chi tiết course có giá > 0
2. Hệ thống hiển thị button "Enroll Now" với giá course
3. Student click "Enroll Now"
4. Hệ thống chuyển hướng đến trang thanh toán
5. Hệ thống hiển thị trang thanh toán:
   - Tiêu đề course
   - Giá gốc
   - Giảm giá (nếu có discount đang áp dụng)
   - Số tiền cuối cùng (final amount)
   - Button "Pay with VNPay"
6. Student click "Pay with VNPay"
7. Hệ thống tạo payment record:
   - user_id = student_id
   - course_id
   - amount = final amount
   - status = "pending"
   - transaction_id = UUID
   - created_at = thời điểm hiện tại
8. Hệ thống chuẩn bị VNPay request:
   - vnp_Amount = amount × 100 (VNPay yêu cầu đơn vị cent)
   - vnp_OrderInfo = "Payment for [Course Title]"
   - vnp_TxnRef = transaction_id
   - vnp_ReturnUrl = callback URL của platform
9. Hệ thống tạo secure hash signature theo VNPay specification
10. Hệ thống chuyển hướng Student đến VNPay payment gateway với parameters
11. Student hoàn tất thanh toán trên VNPay:
    - Chọn phương thức: Thẻ ATM, Thẻ tín dụng, QR Code, Ví điện tử
    - Xác thực thanh toán
12. VNPay xử lý giao dịch
13. VNPay callback về vnp_ReturnUrl với kết quả:
    - vnp_ResponseCode
    - vnp_TransactionNo
    - vnp_SecureHash
14. Hệ thống verify VNPay signature để đảm bảo tính toàn vẹn
15. Hệ thống parse vnp_ResponseCode:
    - Nếu "00": Thanh toán thành công → chuyển sang bước 16
    - Nếu khác "00": Thanh toán thất bại → UC-07.03 AF1
16. Hệ thống cập nhật payment record:
    - status = "success"
    - vnpay_transaction_no = vnp_TransactionNo
    - paid_at = thời điểm hiện tại
17. Hệ thống tạo enrollment record:
    - user_id = student_id
    - course_id
    - enrolled_at = thời điểm hiện tại
    - payment_status = "paid"
    - payment_id = payment record ID
18. Hệ thống tăng course.enrollment_count += 1
19. Hệ thống gửi email xác nhận:
    - Subject: "Thanh toán thành công - [Course Name]"
    - Body: Thông tin course, invoice, link truy cập
20. Hệ thống chuyển hướng Student đến trang thành công với thông tin:
    - "Thanh toán thành công!"
    - Course đã mua
    - Button "Start Learning"
21. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Thanh toán thất bại**

  - Tại bước 15: vnp_ResponseCode ≠ "00"
  - Hệ thống cập nhật payment record: status = "failed", failure_reason
  - Hệ thống chuyển hướng đến trang thất bại
  - Hệ thống hiển thị thông báo lỗi dựa trên response code:
    - "Số dư không đủ"
    - "Giao dịch bị từ chối"
    - "Thông tin thẻ không hợp lệ"
  - Hệ thống hiển thị options:
    - "Thử lại" → quay lại bước 4
    - "Liên hệ hỗ trợ"
  - Kết thúc Use Case

- **AF2: VNPay signature không hợp lệ**

  - Tại bước 14: Signature verification thất bại
  - Hệ thống ghi log security incident
  - Hệ thống cập nhật payment: status = "failed", reason = "Invalid signature"
  - Hiển thị lỗi: "Giao dịch không hợp lệ. Vui lòng liên hệ support."
  - Kết thúc Use Case

- **AF3: Student hủy thanh toán trên VNPay**

  - Tại bước 11: Student click "Cancel" trên VNPay gateway
  - VNPay callback với response code khác "00"
  - Hệ thống cập nhật payment: status = "cancelled"
  - Chuyển hướng về trang course detail
  - Hiển thị: "Bạn đã hủy thanh toán"
  - Kết thúc Use Case

- **AF4: Timeout khi chờ VNPay callback**

  - Tại bước 13: Không nhận được callback sau 10 phút
  - Hệ thống đánh dấu payment: status = "pending_verification"
  - Student có thể kiểm tra trạng thái sau trong Payment History
  - Admin có thể verify thủ công
  - Kết thúc Use Case

- **AF5: Student đã enroll trong lúc thanh toán**
  - Tại bước 17: Enrollment đã tồn tại (race condition)
  - Hệ thống skip tạo enrollment mới (idempotency)
  - Vẫn cập nhật payment = success
  - Hiển thị: "Thanh toán thành công. Bạn đã có quyền truy cập course."
  - Kết thúc Use Case

**Business Rules:** BR-05
