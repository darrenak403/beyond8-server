# Use Case-08.03: Mark Review as Helpful

**Use Case ID:** UC-08.03  
**Module:** Rating & Review  
**Actor chính:** Student, Guest  
**Actor phụ:** Không có

**Mô tả ngắn:** User đánh dấu review là hữu ích

**Trigger:** User nhấn button "Helpful" dưới review

**Pre-condition:**

- Review tồn tại và visible
- User chưa đánh dấu review này là helpful (hoặc đã đánh dấu và muốn bỏ)

**Post-condition:**

- Trạng thái helpful được cập nhật
- Số lượt helpful được tăng/giảm

**Main Flow:**

1. User xem danh sách reviews (UC-08.02)
2. Hệ thống hiển thị button "Helpful" dưới mỗi review
3. Hệ thống hiển thị số lượt helpful hiện tại: "X người thấy hữu ích"
4. User nhấn "Helpful"
5. Hệ thống kiểm tra:
   - Nếu Guest: sử dụng session ID hoặc IP để tracking
   - Nếu Student: sử dụng user_id
6. Hệ thống kiểm tra user đã đánh dấu helpful cho review này chưa:
   - Nếu chưa → Thêm helpful mark (tăng count)
   - Nếu rồi → Xóa helpful mark (giảm count)
7. Hệ thống cập nhật bảng review_helpful:
   - Nếu chưa có: INSERT (user_id/session_id, review_id, created_at)
   - Nếu đã có: DELETE
8. Hệ thống cập nhật helpful_count trong bảng reviews:
   - Tăng hoặc giảm 1
9. Hệ thống cập nhật UI ngay lập tức:
   - Toggle trạng thái button (active/inactive)
   - Update số lượt: "X người thấy hữu ích"
10. Kết thúc Use Case

**Alternate Flow:**

- **AF1: User chưa đăng nhập (Guest)**

  - Tại bước 5: User là Guest
  - Hệ thống lưu helpful mark với session_id hoặc IP
  - Helpful mark có thể bị mất khi clear session/cookies
  - Tiếp tục bước 7

- **AF2: Spam detection**

  - Tại bước 6: User (Guest) đã đánh dấu quá nhiều reviews trong thời gian ngắn
  - Hệ thống rate limit: tối đa 10 helpful marks mỗi 5 phút
  - Hiển thị: "Vui lòng thử lại sau"
  - Kết thúc Use Case

- **AF3: Review đã bị xóa**
  - Tại bước 4: Review có status = deleted
  - Hệ thống không cho phép helpful
  - Button "Helpful" bị disabled
  - Kết thúc Use Case
