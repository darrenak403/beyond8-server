# Use Case-08.04: Report Inappropriate Review

**Use Case ID:** UC-08.04  
**Module:** Rating & Review  
**Actor chính:** Instructor, Student  
**Actor phụ:** Admin (xử lý report)

**Mô tả ngắn:** Report reviews vi phạm quy định

**Trigger:** User nhấn "Report" dưới review

**Pre-condition:**

- User đã đăng nhập
- Review tồn tại và visible

**Post-condition:**

- Report được ghi nhận với status = pending
- Admin nhận notification để xem xét

**Main Flow:**

1. User xem review trên course detail page
2. Hệ thống hiển thị link/icon "Report" dưới review
3. User nhấn "Report"
4. Hệ thống hiển thị form report:
   - Dropdown "Lý do":
     - Spam
     - Inappropriate language
     - Fake review
     - Offensive content
   - Text area "Chi tiết bổ sung" (tùy chọn, tối đa 300 ký tự)
   - Button "Submit Report"
5. User chọn lý do
6. User nhập chi tiết bổ sung (tùy chọn)
7. User nhấn "Submit Report"
8. Hệ thống xác thực:
   - Lý do không trống
   - Chi tiết ≤ 300 ký tự
9. Hệ thống tạo bản ghi review_report:
   - reporter_id (user_id)
   - review_id
   - reason
   - details
   - status = "pending"
   - created_at
10. Hệ thống gửi notification cho Admin
11. Hệ thống hiển thị: "Cảm ơn! Report của bạn đã được ghi nhận"
12. Hệ thống ẩn button "Report" cho review này (với user này)
13. Kết thúc Use Case

**Alternate Flow:**

- **AF1: User đã report review này**

  - Tại bước 3: User đã report review này trước đó
  - Button "Report" bị disabled hoặc hiển thị "Reported"
  - Hiển thị tooltip: "Bạn đã report review này"
  - Kết thúc Use Case

- **AF2: Chi tiết quá dài**

  - Tại bước 8: Chi tiết > 300 ký tự
  - Hệ thống hiển thị lỗi: "Chi tiết không vượt quá 300 ký tự"
  - Quay lại bước 6

- **AF3: Admin xử lý report - Xóa review**

  - Admin xem xét report
  - Admin quyết định xóa review
  - Hệ thống soft delete review (status = deleted)
  - Review ẩn khỏi công khai
  - Hiển thị placeholder: "Review này đã bị xóa vì vi phạm quy định"
  - Gửi notification cho reviewer
  - Cập nhật status report = "resolved"

- **AF4: Admin xử lý report - Cảnh báo**

  - Admin xem xét report
  - Admin quyết định cảnh báo reviewer
  - Hệ thống gửi email cảnh báo cho reviewer
  - Ghi nhận warning vào user profile
  - Review vẫn giữ nguyên
  - Cập nhật status report = "resolved"

- **AF5: Admin xử lý report - Bỏ qua**
  - Admin xem xét report
  - Admin quyết định bỏ qua (report không hợp lệ)
  - Cập nhật status report = "dismissed"
  - Review vẫn giữ nguyên
