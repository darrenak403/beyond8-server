# Use Case-01.06: Update Profile

**Use Case ID:** UC-01.06

**Module:** User Management

**Actor chính:** Học Viên, Giảng Viên, Quản Trị Viên

**Actor phụ:** Không có

**Mô tả ngắn:** User cập nhật thông tin cá nhân

**Trigger:** User điều hướng đến trang "Edit Profile"

**Pre-condition:**

- User đã đăng nhập

**Post-condition:**

- Hồ sơ người dùng đã được cập nhật
- Thay đổi có hiệu lực ngay lập tức

**Main Flow:**

1. User nhấn nút "Chỉnh Sửa Hồ Sơ" trên trang hồ sơ
2. Hệ thống tải dữ liệu hồ sơ hiện tại vào form (full name, phone, tiểu sử, avatar)
3. User chỉnh sửa các trường được phép:

   - Full name
   - Phone
   - Tiểu sử (tối đa 500 ký tự)
   - Ảnh avatar (tải lên file mới)

4. Nếu avatar được tải lên:

   - Hệ thống xác thực: tối đa 5MB, định dạng JPG/PNG
   - Hệ thống tải lên S3
   - Hệ thống tự động resize về 300x300px
   - Hệ thống tạo URL cho avatar

5. Hệ thống xác thực tất cả các trường
6. Hệ thống cập nhật bản ghi user trong database
7. Hệ thống hiển thị thông báo thành công: "Cập nhật hồ sơ thành công"
8. Hệ thống làm mới trang hồ sơ với dữ liệu mới
9. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Avatar file too large**

  - Tại bước 4: Kích thước file > 5MB
  - Hệ thống hiển thị lỗi: "Ảnh avatar phải nhỏ hơn 5MB"
  - Quay lại bước 3

- **AF2: Invalid avatar format**

  - Tại bước 4: Định dạng file không phải JPG/PNG
  - Hệ thống hiển thị lỗi: "Vui lòng tải lên ảnh định dạng JPG hoặc PNG"
  - Quay lại bước 3

- **AF3: Bio too long**

  - Tại bước 5: Độ dài tiểu sử > 500 ký tự
  - Hệ thống hiển thị lỗi: "Tiểu sử tối đa là 500 ký tự"
  - Quay lại bước 3

- **AF4: User attempts to change email/role**
  - Tại bước 3: User cố gắng chỉnh sửa các trường chỉ đọc (email, role)
  - Hệ thống bỏ qua thay đổi (các trường bị vô hiệu hóa trong UI)
  - Tiếp tục đến bước 5

**Business Rules:** Không có quy tắc cụ thể
