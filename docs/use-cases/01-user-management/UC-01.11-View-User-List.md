# Use Case-01.11: View User List (Admin)

**Use Case ID:** UC-01.11

**Module:** User Management

**Actor chính:** Quản trị viên (Admin)

**Actor phụ:** Không có

**Mô tả ngắn:** Admin xem danh sách tất cả người dùng trong hệ thống

**Trigger:** Admin điều hướng đến trang "Quản lý người dùng" (Manage Users)

**Pre-condition:**

- Admin đã đăng nhập

**Post-condition:**

- Danh sách người dùng được hiển thị kèm theo các bộ lọc/tìm kiếm đã được áp dụng

**Main Flow:**

1. Admin nhấn "Quản Lý Người Dùng" trong thanh bên của bảng điều khiển admin
2. Hệ thống tải danh sách người dùng có phân trang (50 người dùng/trang)
3. Hệ thống hiển thị bảng với các cột: Email, Tên, Vai Trò, Trạng Thái, Ngày Đăng Ký, Đăng Nhập Lần Cuối, Hành Động
4. Admin có thể thực hiện:

   - Tìm kiếm theo email hoặc tên (lọc theo thời gian thực)
   - Lọc theo vai trò (Student, Instructor, Admin)
   - Lọc theo trạng thái (Active, Banned, PendingVerification)
   - Sắp xếp theo: ngày đăng ký, lần đăng nhập cuối cùng

5. Admin có thể thực hiện hành động nhanh cho từng người dùng:

   - Xem Hồ Sơ
   - Thay Đổi Vai Trò
   - Cấm/Mở Cấm
   - Xóa (xóa mềm - soft delete)

6. Use Case kết thúc khi Admin rời khỏi trang

**Alternate Flow:**

- **AF1: Search no results**

  - Tại bước 4: Truy vấn tìm kiếm không khớp với bất kỳ người dùng nào
  - Hệ thống hiển thị: "Không tìm thấy người dùng phù hợp với '[truy vấn]'"
  - Admin có thể xóa tìm kiếm để xem toàn bộ danh sách

- **AF2: Export user list**
  - Tại bước 4: Admin nhấn nút "Export CSV"
  - Hệ thống tạo tệp CSV dựa trên các bộ lọc hiện đang áp dụng
  - Hệ thống thực hiện tải tệp xuống
  - Quay lại bước 4

**Business Rules:** Không có quy tắc cụ thể
