# Use Case-01.03: User Login

**Use Case ID:** UC-01.03

**Module:** User Management

**Actor chính:** Student, Instructor, Admin

**Actor phụ:** Không có

**Mô tả ngắn:** Người dùng đã đăng ký đăng nhập vào hệ thống

**Trigger:** User nhấn nút "Login"

**Pre-condition:**

- User có tài khoản đã xác thực
- User chưa đăng nhập

**Post-condition:**

- User được xác thực thành công
- JWT token được cấp
- User được chuyển hướng đến dashboard phù hợp với vai trò

**Main Flow:**

1. User nhập email và mật khẩu
2. Hệ thống xác thực thông tin đăng nhập:

   - Email tồn tại trong database
   - Hash mật khẩu khớp với hash đã lưu

3. Hệ thống kiểm tra trạng thái user:

   - Trạng thái = "Active" (không phải "Banned" hay "PendingVerification")

4. Hệ thống tạo JWT token (payload: user_id, role, email, hết hạn sau 24h)
5. Hệ thống cập nhật thời gian last_login_at
6. Hệ thống chuyển hướng dựa trên vai trò:

   - Student → Student Dashboard
   - Instructor → Instructor Dashboard
   - Admin → Admin Panel

7. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Invalid credentials**

  - Tại bước 2: Không tìm thấy email HOẶC mật khẩu không khớp
  - Hệ thống hiển thị lỗi chung: "Email hoặc mật khẩu không hợp lệ" (bảo mật: không tiết lộ thông tin nào sai)
  - Tăng biến đếm failed_login_attempts
  - Quay lại bước 1

- **AF2: Account not verified**

  - Tại bước 3: Trạng thái user = "PendingVerification"
  - Hệ thống hiển thị: "Vui lòng xác thực email trước khi đăng nhập"
  - Cung cấp nút "Resend Verification Email"
  - Kết thúc Use Case

- **AF3: Account banned**

  - Tại bước 3: Trạng thái user = "Banned"
  - Hệ thống hiển thị: "Tài khoản của bạn đã bị tạm ngưng. Vui lòng liên hệ hỗ trợ để biết chi tiết."
  - Kết thúc Use Case

- **AF4: Too many failed attempts**
  - Tại AF1: Nếu failed_login_attempts ≥ 5 lần trong vòng 15 phút
  - Hệ thống khóa tài khoản trong 15 phút
  - Hệ thống hiển thị: "Quá nhiều lần thử thất bại. Vui lòng thử lại sau 15 phút."
  - Kết thúc Use Case
