# Use Case-01.09: Change User Role (Admin)

**Use Case ID:** UC-01.09

**Module:** User Management

**Actor chính:** Quản trị viên (Admin)

**Actor phụ:** User đích (được thông báo)

**Mô tả ngắn:** Admin thay đổi vai trò (role) của người dùng

**Trigger:** Admin chọn "Change Role" trên trang quản lý người dùng

**Pre-condition:**

- Admin đã đăng nhập
- User đích tồn tại trong hệ thống
- Admin không thể tự thay đổi vai trò của chính mình

**Post-condition:**

- Vai trò của người dùng đã được cập nhật
- User đích nhận được thông báo
- Thay đổi đã được ghi lại

**Main Flow:**

1. Admin điều hướng đến trang "Quản Lý Người Dùng"
2. Admin tìm kiếm user đích (theo email hoặc tên)
3. Admin nhấn nút "Thay Đổi Vai Trò" bên cạnh người dùng
4. Hệ thống hiển thị vai trò hiện tại và dropdown với các tùy chọn: Student, Instructor
5. Admin chọn vai trò mới
6. Hệ thống xác nhận: "Thay đổi vai trò của [Tên User] từ [Vai Trò Hiện Tại] sang [Vai Trò Mới]?"
7. Admin xác nhận
8. Hệ thống cập nhật vai trò của người dùng trong database
9. Hệ thống ghi lại hành động (admin_id, user_id, old_role, new_role, timestamp)
10. Hệ thống gửi email cho người dùng: "Vai trò tài khoản của bạn đã được cập nhật thành [Vai Trò Mới]"
11. Hệ thống hiển thị thông báo thành công cho Admin
12. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Downgrade Instructor to Student**

  - Tại bước 5: Admin chọn "Student" cho một Instructor
  - Hệ thống cảnh báo: "Hạ cấp xuống Student sẽ ẩn tất cả các khóa học đã xuất bản của giảng viên này. Tiếp tục?"
  - Nếu Admin xác nhận:
  - Hệ thống thay đổi vai trò thành Student
  - Hệ thống chuyển trạng thái tất cả các khóa học của giảng viên đó thành "Unpublished"
  - Hệ thống thông báo cho giảng viên
  - Nếu Admin hủy → quay lại bước 5

- **AF2: Admin attempts to change own role**

  - Tại bước 3: User đích là chính admin đang đăng nhập
  - Hệ thống chặn hành động: "Bạn không thể tự thay đổi vai trò của chính mình"
  - Kết thúc Use Case

- **AF3: Admin cancels confirmation**
  - Tại bước 7: Admin nhấn "Hủy"
  - Không có thay đổi nào được thực hiện
  - Kết thúc Use Case

**Business Rules:** Không có quy tắc cụ thể
