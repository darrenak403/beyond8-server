# Use Case-01.02: Email Verification

**Use Case ID:** UC-01.02

**Module:** User Management

**Actor chính:** Guest User

**Actor phụ:** Không có

**Mô tả ngắn:** User xác thực email để kích hoạt tài khoản

**Trigger:** User nhập mã OTP xác thực

**Pre-condition:**

- User đã đăng ký (UC-01.01 hoàn tất)
- OTP chưa hết hạn

**Post-condition:**

- Trạng thái tài khoản User = "Active"
- User được tự động đăng nhập
- JWT token được cấp

**Main Flow:**

1. User nhập mã OTP 6 chữ số từ email
2. Hệ thống xác thực OTP:

   - OTP khớp với mã đã gửi
   - OTP chưa hết hạn (<5 phút)
   - Số lần thử không vượt quá 5 lần

3. Hệ thống cập nhật trạng thái user thành "Active"
4. Hệ thống xóa OTP khỏi database
5. Hệ thống tạo JWT token (hiệu lực 24 giờ)
6. Hệ thống chuyển hướng user đến dashboard
7. Hệ thống hiển thị thông báo chào mừng: "Email đã được xác thực! Chào mừng bạn đến với [Tên nền tảng]"
8. Kết thúc Use Case

**Alternate Flow:**

- **AF1: OTP hết hạn**

  - Tại bước 2: Thời gian tạo OTP >5 phút trước
  - Hệ thống hiển thị lỗi: "Mã OTP đã hết hạn"
  - Hệ thống hiển thị nút "Gửi lại OTP"
  - Nếu nhấn → tạo OTP mới và gửi email → quay lại bước 1

- **AF2: OTP không đúng**

  - Tại bước 2: OTP nhập vào không khớp
  - Hệ thống tăng số lần thử lên 1
  - Hệ thống hiển thị lỗi: "Mã OTP không đúng. Bạn còn X lần thử."
  - Quay lại bước 1

- **AF3: Vượt quá số lần thử**

  - Tại bước 2: Số lần thử > 5
  - Hệ thống vô hiệu hóa OTP hiện tại
  - Hệ thống hiển thị: "Bạn đã nhập sai quá nhiều lần. Vui lòng yêu cầu mã OTP mới."
  - Hiển thị nút "Gửi lại OTP"

- **AF4: Email đã được xác thực**
  - Tại bước 2: Trạng thái user đã là "Active"
  - Hệ thống hiển thị: "Email đã được xác thực. Vui lòng đăng nhập."
  - Chuyển hướng đến trang đăng nhập
  - Kết thúc Use Case
