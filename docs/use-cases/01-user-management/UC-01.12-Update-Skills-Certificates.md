# Use Case-01.12: Update Skills and Certificates

**Use Case ID:** UC-01.12

**Module:** User Management

**Actor chính:** Học Viên, Giảng Viên

**Actor phụ:** Không có

**Mô tả ngắn:** Người dùng quản lý các kỹ năng và chứng chỉ trên hồ sơ cá nhân

**Trigger:** User điều hướng đến trang hồ sơ cá nhân (Profile)

**Pre-condition:**

- User đã đăng nhập

**Post-condition:**

- Các kỹ năng được cập nhật trên hồ sơ
- Các chứng chỉ hoàn thành khóa học được tự động hiển thị

**Main Flow:**

1. User điều hướng đến trang hồ sơ của mình
2. Hệ thống hiển thị:

   - Phần Kỹ năng (dưới dạng các thẻ/tags, có thể chỉnh sửa)
   - Phần Chứng chỉ (tự động điền dữ liệu từ các khóa học đã hoàn thành)

3. User thêm hoặc xóa các kỹ năng (tối đa là 20 kỹ năng)
4. User nhấn nút "Lưu Kỹ Năng"
5. Hệ thống xác thực: số lượng kỹ năng ≤ 20
6. Hệ thống cập nhật hồ sơ người dùng trong hệ thống
7. Hệ thống hiển thị thông báo thành công
8. Đối với chứng chỉ:

   - Hệ thống tự động thêm chứng chỉ khi người dùng hoàn thành một khóa học (theo UC-06.05)
   - Người dùng không thể tự ý thêm hoặc xóa các chứng chỉ này để đảm bảo tính xác thực

9. User xem hồ sơ dưới góc nhìn công khai → các kỹ năng và chứng chỉ sẽ được hiển thị
10. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Too many skills**

  - Tại bước 5: Số lượng kỹ năng vượt quá 20
  - Hệ thống hiển thị lỗi: "Bạn chỉ có thể thêm tối đa 20 kỹ năng"
  - Quay lại bước 3

- **AF2: View other's profile**
  - Tại bước 1: User điều hướng đến trang hồ sơ của một người khác
  - Hệ thống hiển thị hồ sơ ở chế độ chỉ đọc (không có nút chỉnh sửa hay lưu)
  - Các kỹ năng và chứng chỉ vẫn được hiển thị công khai
  - Kết thúc Use Case
