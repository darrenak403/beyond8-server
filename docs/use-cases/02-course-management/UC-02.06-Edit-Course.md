# Use Case-02.02: Chỉnh Sửa Thông Tin Khóa Học

**Use Case ID:** UC-02.02

**Module:** Quản lý khóa học (Course Management)

**Actor chính:** Giảng viên (Instructor)

**Actor phụ:** Không có

**Mô tả ngắn:** Giảng viên cập nhật thông tin khóa học

**Trigger:** Giảng viên nhấn "Chỉnh sửa khóa học" (Edit Course) trên bảng điều khiển khóa học

**Pre-condition:**

- Giảng viên sở hữu khóa học này
- Trạng thái khóa học = "Draft" (Bản nháp) hoặc "Rejected" (Bị từ chối). (Không thể chỉnh sửa nếu ở trạng thái "PendingApproval" hoặc "Published")

**Post-condition:**

- Thông tin khóa học được cập nhật
- Lịch sử phiên bản (version history) được lưu lại

**Main Flow:**

1. Giảng viên điều hướng đến danh sách khóa học của mình
2. Giảng viên nhấn "Chỉnh sửa" trên khóa học muốn sửa
3. Hệ thống kiểm tra trạng thái khóa học (chỉ cho phép chỉnh sửa Draft/Rejected)
4. Hệ thống tải dữ liệu hiện tại vào biểu mẫu
5. Giảng viên chỉnh sửa các trường (tương tự UC-02.01 bước 3-4)
6. Hệ thống xác thực dữ liệu đầu vào
7. Hệ thống lưu lịch sử phiên bản (ảnh chụp thông tin cũ kèm dấu thời gian)
8. Hệ thống cập nhật bản ghi khóa học
9. Hệ thống hiển thị: "Thay đổi đã được lưu"
10. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Khóa học đang chờ duyệt**

  - Tại bước 3: Trạng thái khóa học = "PendingApproval"
  - Hệ thống hiển thị: "Không thể chỉnh sửa khóa học đang chờ duyệt"
  - Kết thúc Use Case

- **AF2: Khóa học đã xuất bản**

  - Tại bước 3: Trạng thái khóa học = "Published"
  - Hệ thống hiển thị: "Không thể chỉnh sửa khóa học đã xuất bản. Vui lòng gỡ xuất bản trước."
  - Cung cấp liên kết đến UC-02.10 (Gỡ xuất bản/Unpublish)
  - Kết thúc Use Case

- **AF3: Không có thay đổi nào**
  - Tại bước 6: Tất cả các trường giống hệt với giá trị hiện tại
  - Hệ thống hiển thị: "Không có thay đổi để lưu"
  - Không tạo lịch sử phiên bản mới
  - Kết thúc Use Case
