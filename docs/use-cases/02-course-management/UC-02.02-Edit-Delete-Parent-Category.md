# Use Case-02.02: Chỉnh Sửa/Xóa Hạng Mục Lớn

**Use Case ID:** UC-02.02

**Module:** Quản lý khóa học (Course Management)

**Actor chính:** Quản trị viên (Admin)

**Actor phụ:** Không có

**Mô tả ngắn:** Admin chỉnh sửa hoặc xóa hạng mục lớn

**Trigger:** Admin nhấn nút "Chỉnh sửa" hoặc "Xóa" trên hạng mục lớn

**Pre-condition:**

- Người dùng đã đăng nhập với vai trò = Admin
- Hạng mục lớn đã tồn tại trong hệ thống

**Post-condition:**

- **Chỉnh sửa:** Thông tin hạng mục được cập nhật
- **Xóa:** Hạng mục được soft delete (ẩn khỏi hệ thống)

**Main Flow - Chỉnh Sửa:**

1. Admin điều hướng đến trang "Quản lý danh mục"
2. Admin nhấn nút "Chỉnh sửa" trên hạng mục lớn cần sửa
3. Hệ thống hiển thị biểu mẫu với dữ liệu hiện tại
4. Admin chỉnh sửa các trường:

   - Tên hạng mục
   - Mô tả
   - Icon
   - Màu sắc
   - Banner

5. Admin nhấn "Lưu thay đổi"
6. Hệ thống xác thực dữ liệu
7. Hệ thống cập nhật thông tin với updated_at = thời gian hiện tại
8. Hệ thống hiển thị thông báo: "Cập nhật thành công!"
9. Kết thúc Use Case

**Main Flow - Xóa:**

1. Admin điều hướng đến trang "Quản lý danh mục"
2. Admin nhấn nút "Xóa" trên hạng mục lớn cần xóa
3. Hệ thống kiểm tra:

   - Có hạng mục con nào thuộc hạng mục này không?
   - Có khóa học nào liên kết với hạng mục này không?

4. Nếu không có hạng mục con và khóa học → hiển thị dialog xác nhận
5. Dialog hiển thị: "Bạn có chắc muốn xóa hạng mục '[Tên hạng mục]'? Hành động này không thể hoàn tác."
6. Admin nhấn "Xác nhận"
7. Hệ thống thực hiện soft delete:

   - status = "Deleted"
   - deleted_at = thời gian hiện tại
   - deleted_by = admin hiện tại

8. Hệ thống hiển thị thông báo: "Hạng mục đã được xóa!"
9. Hạng mục biến mất khỏi danh sách
10. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Hạng mục có hạng mục con**

  - Tại bước 3 (Xóa): Hệ thống phát hiện có hạng mục con
  - Hệ thống hiển thị lỗi: "Không thể xóa! Hạng mục này có [X] hạng mục con. Vui lòng xóa hoặc di chuyển các hạng mục con trước."
  - Kết thúc Use Case

- **AF2: Hạng mục có khóa học liên kết**

  - Tại bước 3 (Xóa): Hệ thống phát hiện có khóa học
  - Hệ thống hiển thị lỗi: "Không thể xóa! Hạng mục này có [Y] khóa học. Vui lòng di chuyển các khóa học sang hạng mục khác trước."
  - Kết thúc Use Case

- **AF3: Tên trùng lặp khi chỉnh sửa**

  - Tại bước 6 (Chỉnh sửa): Tên mới trùng với hạng mục khác
  - Hệ thống hiển thị lỗi: "Tên hạng mục đã tồn tại. Vui lòng chọn tên khác."
  - Quay lại bước 4

- **AF4: Admin hủy xóa**
  - Tại bước 6 (Xóa): Admin nhấn "Hủy" trong dialog
  - Không có thay đổi nào được thực hiện
  - Kết thúc Use Case
