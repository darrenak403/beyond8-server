# Use Case-02.04: Chỉnh Sửa/Xóa Hạng Mục Con

**Use Case ID:** UC-02.04

**Module:** Quản lý khóa học (Course Management)

**Actor chính:** Quản trị viên (Admin)

**Actor phụ:** Không có

**Mô tả ngắn:** Admin chỉnh sửa hoặc xóa hạng mục con

**Trigger:** Admin nhấn nút "Chỉnh sửa" hoặc "Xóa" trên hạng mục con

**Pre-condition:**

- Người dùng đã đăng nhập với vai trò = Admin
- Hạng mục con đã tồn tại trong hệ thống

**Post-condition:**

- **Chỉnh sửa:** Thông tin hạng mục con được cập nhật
- **Xóa:** Hạng mục con được soft delete

**Main Flow - Chỉnh Sửa:**

1. Admin điều hướng đến trang "Quản lý danh mục"
2. Admin mở rộng cây phân cấp và nhấn "Chỉnh sửa" trên hạng mục con
3. Hệ thống hiển thị biểu mẫu với dữ liệu hiện tại
4. Admin chỉnh sửa các trường:

   - Tên hạng mục con
   - Hạng mục lớn cha (có thể di chuyển sang hạng mục khác)
   - Mô tả
   - Icon
   - Thứ tự hiển thị

5. Admin nhấn "Lưu thay đổi"
6. Hệ thống xác thực dữ liệu
7. Hệ thống kiểm tra tên unique trong hạng mục cha mới (nếu có thay đổi)
8. Hệ thống cập nhật thông tin với updated_at = thời gian hiện tại
9. Nếu hạng mục cha bị thay đổi → cập nhật parent_category_id
10. Hệ thống hiển thị thông báo: "Cập nhật thành công!"
11. Cập nhật cây phân cấp danh mục
12. Kết thúc Use Case

**Main Flow - Xóa:**

1. Admin điều hướng đến trang "Quản lý danh mục"
2. Admin mở rộng cây phân cấp và nhấn "Xóa" trên hạng mục con
3. Hệ thống kiểm tra: Có khóa học nào liên kết với hạng mục con này không?
4. Nếu không có khóa học → hiển thị dialog xác nhận
5. Dialog hiển thị: "Bạn có chắc muốn xóa hạng mục con '[Tên hạng mục]'? Hành động này không thể hoàn tác."
6. Admin nhấn "Xác nhận"
7. Hệ thống thực hiện soft delete:

   - status = "Deleted"
   - deleted_at = thời gian hiện tại
   - deleted_by = admin hiện tại

8. Hệ thống hiển thị thông báo: "Hạng mục con đã được xóa!"
9. Cập nhật cây phân cấp (hạng mục con biến mất)
10. Kết thúc Use Case

**Alternate Flow:**

- **AF1: Hạng mục con có khóa học liên kết**

  - Tại bước 3 (Xóa): Hệ thống phát hiện có khóa học
  - Hệ thống hiển thị lỗi: "Không thể xóa! Hạng mục con này có [X] khóa học. Vui lòng di chuyển các khóa học sang hạng mục khác trước."
  - Hiển thị danh sách các khóa học liên kết
  - Kết thúc Use Case

- **AF2: Tên trùng lặp khi chỉnh sửa**

  - Tại bước 7 (Chỉnh sửa): Tên mới trùng với hạng mục con khác trong cùng hạng mục cha
  - Hệ thống hiển thị lỗi: "Tên hạng mục con đã tồn tại trong hạng mục cha này."
  - Quay lại bước 4

- **AF3: Di chuyển sang hạng mục cha đã có tên trùng**

  - Tại bước 7 (Chỉnh sửa): Hạng mục cha mới đã có hạng mục con cùng tên
  - Hệ thống hiển thị lỗi: "Hạng mục cha '[Tên hạng mục cha mới]' đã có hạng mục con tên '[Tên]'. Vui lòng đổi tên hoặc chọn hạng mục cha khác."
  - Quay lại bước 4

- **AF4: Admin hủy xóa**
  - Tại bước 6 (Xóa): Admin nhấn "Hủy" trong dialog
  - Không có thay đổi nào được thực hiện
  - Kết thúc Use Case
